COMANDOS GIT

Comando utilizado para salvar a primeira versão de um projeto no GitHub:

  1. git init: Este comando é utilizado para iniciar um novo repositório Git. Ele cria um novo subdiretório no seu sistema de arquivos local com um repositório do Git.
    git init
  1. git add .: Este comando adiciona todas as mudanças no diretório de trabalho ao índice (staging area). O ponto (.) representa todos os arquivos no diretório atual.
    git add .
  1. git commit -m "Mensagem explicativa": Este comando salva as mudanças que foram adicionadas ao índice, criando uma nova versão do seu projeto. A opção m permite adicionar uma mensagem explicativa diretamente na linha de comando.
    git commit -m "Mensagem explicativa"
  1. git branch -M main: Este comando renomeia a branch padrão de master para main. O GitHub, por padrão, usa main como o nome da branch principal.
    git branch -M main
  1. git remote add origin github.com:seuusuario/seurepositorio.git: Este comando adiciona um repositório remoto chamado origin ao seu projeto Git. O endereço fornecido é o URL do seu repositório no GitHub.
    git remote add origin github.com:seuusuario/seurepositorio.git
  1. git push -u origin main: Este comando envia as mudanças locais para o repositório remoto (no GitHub, neste caso). A opção u configura o branch local como o branch padrão para o branch remoto, o que significa que, nas próximas vezes que você fizer push, você pode simplesmente usar git push sem especificar a branch.
    git push -u origin main

Após executar esses comandos, sua primeira versão do projeto será enviada para o GitHub. Certifique-se de substituir seuusuario e seurepositorio pelos seus dados reais do GitHub.

git pull e git clone são dois comandos distintos no Git, e eles têm propósitos diferentes:

  1. git clone: Este comando é utilizado para criar uma cópia local de um repositório remoto. Quando você clona um repositório, o Git cria uma cópia de todos os arquivos no repositório remoto e configura automaticamente o repositório local para rastrear o remoto. A sintaxe básica é:
    bashCopy code
    git clone <URL_do_repositorio_remoto>

    Por exemplo:

    bashCopy code
    git clone https://github.com/exemplo/um-repositorio.git

    Isso cria um diretório chamado um-repositorio no seu sistema de arquivos local, com todos os arquivos do repositório remoto.

  1. git pull: Este comando é utilizado para atualizar o seu repositório local com as alterações mais recentes do repositório remoto. Ele é especialmente útil quando outras pessoas contribuíram para o mesmo projeto e você deseja sincronizar seu repositório local com as alterações mais recentes. A sintaxe básica é:
    bashCopy code
    git pull <remoto> <branch>

    Por exemplo:

    bashCopy code
    git pull origin master

    Este comando traz as alterações da branch master do repositório remoto chamado origin para a sua branch local.

Resumindo, enquanto git clone é usado para criar uma cópia local completa de um repositório remoto, git pull é usado para atualizar sua cópia local com as alterações mais recentes do repositório remoto, mantendo-a sincronizada.
